GITHUB EXAMPLES

  • View-data binding on Android

Use annotations to bind data to views without any view lookup. Sync all view data in one go and keep view states up to date with application state at all times. View holders are bulit in.

Get started!

Binding data to a view

Data is bound by annotating a field or a method with any of the BindTo-annotaitons with a provided view id. Passing the instance containing the data and a view finder (an activity or a view containing the view) will fulfill the binding.

Custom setter

Most BindTo-annotations are configured with a default setter that will be used when bound. This can be altered by specifying a custom set-paramter. A custom set-paramter must be backed up by a public setter on the view type used.

Custom binder

For more advanced bindings, a custom binder can be provided with the @Binds annotation. The binder is mapped to the annotated value by prefixing the the binder property with binds[Property]. In the code sample bindsTitle provides binder for title.

Simple bind control

Witch is designed to bind only values that has changed since last bind. This behaviour can be altered by using the @BindWhen annotation. The default strategy for all values is BindWhen.NOT_EQUALS described below.

Advanced bind control

If non of the bind strategies provided is enough, a Value-container can be used to get full control over when binding should happen.

Bind adapter views

Witch will create view holders for all annotated views. This eliminates the need for creating additional view holders for adapter views. Simply use the RecyclerViewBinderAdapter that requires no implementation besides the actual data binding.

{% highlight java linenos %} {% include snippets/recycler-view-binder.java %} {% endhighlight %}

Since binder will handle binding of all items of type Post in adapter, it's important to separate binder from item. Binder will be updated with a new item before each bind. This happens under the hood and is handled entirely by Witch.

{% highlight java linenos %} {% include snippets/post-binder.java %} {% endhighlight %}

If your data set consists of multiple types, simply provide multiple binders to adapter.
